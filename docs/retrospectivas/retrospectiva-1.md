# Retrospectiva del Proyecto – Iteración 1

## ¿Qué salió bien?

Durante el desarrollo del proyecto se lograron varios avances importantes:

- Se pudo **armar la base del proyecto**, organizando correctamente la estructura de carpetas y los componentes iniciales.
- Se completó el **diagrama de clases**, lo que permitió definir las entidades principales y sus relaciones.
- Se desarrollaron las **historias de usuario**, estableciendo claramente las funcionalidades esperadas desde la perspectiva del usuario final.
- Se elaboró el **documento de requisitos**, consolidando las necesidades del sistema y el alcance del proyecto de manera formal.

Estos resultados permitieron construir una base sólida para avanzar con el desarrollo posterior.

## ¿Qué no salió bien?

El principal desafío estuvo relacionado con la **coordinación grupal**:

- La **comunicación dentro del grupo** no funcionó de la mejor manera.
- Al ser un equipo que nunca había trabajado junto antes, surgieron **diferencias en las formas de trabajo y en los enfoques para avanzar**.
- Resultó difícil **encontrar horarios** en los que todos los integrantes pudieran reunirse, lo que generó demoras y complicó la toma de decisiones de forma conjunta.
- La falta de sincronización hizo que en algunos momentos el **avance no fuera uniforme**.

## ¿Qué se puede mejorar?

Para los próximos trabajos se definieron varias áreas de mejora:

- Mejorar la **comunicación interna**, utilizando canales más efectivos y acordando reglas básicas para el intercambio de información.
- **Planificar reuniones** con mayor anticipación, para asegurar mayor participación y evitar retrasos.
- Definir **roles o responsables** para distintas partes del proyecto, de manera que cada integrante sepa claramente qué debe hacer y cuándo.
- Documentar **acuerdos técnicos**, como convenciones de código, ramas de Git, o decisiones de arquitectura, para evitar inconsistencias.

## Aprendizajes

De la experiencia del proyecto se aprendieron varias lecciones importantes:

- Para que un proyecto salga bien, **la organización del equipo** es tan importante como la parte técnica.
- Una **buena comunicación** evita malentendidos, agiliza el trabajo y permite resolver problemas antes de que se vuelvan grandes.
- Trabajar con personas nuevas implica **adaptarse, escuchar y aprender** distintas formas de encarar un problema.
